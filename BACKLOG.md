## Известные баги и проблемы при импорте

### BUG-1: Поле `question` в SEO JSON не совпадает с заголовком MD
**Статус:** Исправлено в инструкциях агента (agents/content/CLAUDE.md)
**Описание:** Агент переформулировал заголовки вопросов в SEO JSON, из-за чего скрипт импорта не мог сопоставить вопросы. Например: MD: `Может ли объект получить доступ к члену класса, объявленному как private?` → JSON: `Может ли объект получить доступ к private-полям класса?`.
**Решение:** Добавлено правило: поле `question` в SEO JSON = точная копия заголовка `##` из `-v2.md`. Скрипт `import-topic.py` дополнен fuzzy matching как fallback.

### BUG-2: 4 вопроса в core-v2.md с пустым ответом
**Статус:** ИСПРАВЛЕНО (2026-05-09)
**Описание:** Вопросы на строках 68, 71, 76, 91 в core-v2.md имеют заголовок, но пустое тело ответа. API отклоняет их (400 Bad Request, @NotBlank).
**Вопросы:**
- `Если вы хотите переопределить equals(), какие условия должны выполняться?`
- `Какая связь между hashCode() и equals()?`
- `Каковы правила переопределения метода Object.hashCode()?`
- `Какие виды исключений в Java вы знаете, чем они отличаются?`

### BUG-3: Теги из SEO не создавались в таблице `tags`
**Статус:** Исправлено в коде (QuestionService.applyRequest)
**Описание:** При импорте вопросов с новыми тегами через API, теги добавлялись в `question_tags`, но не создавались в таблице `tags`. Страницы `/tags/:slug` возвращали 404.
**Решение:** `QuestionService.applyRequest` теперь автоматически создаёт запись в `tags` если тег новый.

### BUG-4: Вопросы с переформулированными заголовками не импортируются
**Статус:** ИСПРАВЛЕНО (2026-05-12)
**Описание:** Скрипт `import-topic.py` сопоставляет вопросы по тексту заголовка. Агент переформулировал заголовки в SEO JSON (убрал/добавил пунктуацию, кавычки «», backticks, изменил формулировки), из-за чего match не находился.
**Решение:** Пакетная синхронизация полей `question` и `schemaOrg.mainEntity.name` во всех SEO JSON с заголовками `##` из -v2.md (source of truth). Python-скрипт с fuzzy matching + ручная проверка ложных матчей.
**Результат:** 364/364 (100%) exact matches по всем 9 темам.
**Исправлено 115+ расхождений по категориям:**
- Убрана лишняя пунктуация (`?`, `.`) в architecture-batch1 (16), cicd-batch1 (14)
- Добавлены backticks для code-элементов в core-batch2..6 (~50)
- Убраны кавычки `«»` в concurrency-batch1..3 (24)
- Убраны grade-комментарии `<!-- grade: -->` из cf-batch1 (15)
- Исправлены переформулировки (core-batch5, core-batch6, cf-batch1)
- Исправлены 2 ошибочных fuzzy-матча: "Error"→"generics", "HDD/SSD"→"TCP/UDP"

**Рекомендация для импорта:** Повторно запустить `import-topic.py` для всех затронутых тем. Ранее пропущенные 32 вопроса теперь должны импортироваться
